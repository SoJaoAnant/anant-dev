# Car goes brr!!  
### A Physics-Based Top-Down Driving Game

---

## 🧠 Overview

**Car goes brr!!** is a top-down driving game focused on simulating realistic vehicle behavior in a simplified 2D environment.  

The project emphasizes **physics-driven movement**, responsive controls, and engaging gameplay, creating a satisfying driving experience where the player can feel acceleration, turning, and momentum.

---

## ⚙️ Tech Stack

- **Unity**
- **C#**
- **Aseprite**

---

## 🚀 How It Was Made

The game was built using Unity’s 2D framework with a strong focus on physics simulation rather than scripted movement.

- Implemented custom car movement using forces and velocity instead of direct position changes  
- Simulated acceleration, braking, and friction to create realistic driving behavior  
- Designed responsive steering mechanics that account for speed and direction  
- Built collision handling to ensure smooth interaction with the environment  
- Created pixel art assets and sprites using Aseprite  

The result is a system where the car feels dynamic and reactive rather than rigid or arcade-like.

---

## 🎯 Key Learnings

- **Physics-based movement feels more natural than direct transforms**  
  Using forces and velocity leads to smoother and more believable motion  

- **Balancing realism and control is crucial**  
  Too much realism can make controls feel heavy, while too little removes depth  

- **Momentum and friction define driving feel**  
  Small tweaks in drag and acceleration significantly change gameplay  

- **Collision handling needs careful tuning**  
  Poor collision responses can break immersion and gameplay flow  

- **Game feel matters as much as mechanics**  
  Responsiveness, feedback, and smoothness are key to player experience
